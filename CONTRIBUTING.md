First off, thank you for considering contributing to COBOLDecoder! It's people like you that make COBOLDecoder such a great tool for the next generation of mainframe engineers.

## How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/davidthegnomad/COBOLDecoder/issues) page to see if it's already been reported.
*   If not, open a new issue with a clear title and description, including steps to reproduce.

### Suggesting Enhancements
*   We're always looking for more "Quests" or new SQL mappings.
*   Open an issue to discuss the enhancement before starting work.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  Maintain the **Terminal Neon** aesthetic in any UI changes.
3.  Ensure your code follows the standard Flutter linting rules.
4.  Issue a Pull Request!

## Design Language
*   **Palette:** Obsidian (#121212), Cyber-Lime (#32CD32), Electric-Purple (#BF40BF).
*   **Fonts:** Outfit (Heads), Inter (Body), JetBrains Mono / Fira Code (Code).

Thank you for being part of the bridge between modern and legacy systems!
